Quais são os 8 tipos primitivos em Java?	<code>boolean</code>, <code>byte</code>, <code>char</code>, <code>short</code>, <code>int</code>, <code>long</code>, <code>float</code>, <code>double</code>	java::tipos-primitivos
Qual a diferença entre <code>int</code> e <code>long</code> em Java?	<code>int</code> é 32 bits (-2 bilhões a 2 bilhões).<br><code>long</code> é 64 bits (muito maior).<br>Literais long precisam do sufixo <code>L</code>: <code>1000L</code>	java::tipos-primitivos
Qual a diferença entre <code>float</code> e <code>double</code>?	<code>float</code>: 32 bits, 7 dígitos de precisão, sufixo <code>f</code> obrigatório.<br><code>double</code>: 64 bits, 15 dígitos de precisão, sem sufixo.	java::tipos-primitivos
Em Java, quantos tipos numéricos existem vs TypeScript?	Java: <b>7 tipos</b> (byte, short, int, long, float, double, char).<br>TypeScript: <b>1 tipo</b> (<code>number</code>).	java::tipos-primitivos
O que é casting implícito em Java?	Conversão automática quando o tipo destino é <b>maior</b> que a origem.<br>Ex: <code>int x = 10; double y = x;</code> (int → double)	java::tipos-primitivos
O que é casting explícito em Java?	Conversão manual quando há risco de perda de dados. Usa parênteses.<br>Ex: <code>double x = 10.5; int y = (int) x;</code> → resultado: 10	java::tipos-primitivos
O que são Wrapper Classes em Java?	Classes que "envolvem" tipos primitivos para usá-los como objetos.<br>Ex: <code>int</code> → <code>Integer</code>, <code>double</code> → <code>Double</code>	java::wrapper-classes
Por que Wrapper Classes são necessárias?	Generics não aceitam primitivos.<br><code>List&lt;int&gt;</code> ❌<br><code>List&lt;Integer&gt;</code> ✅<br>Também permitem valores <code>null</code>.	java::wrapper-classes
O que é autoboxing e unboxing em Java?	<b>Autoboxing</b>: primitivo → wrapper (automático).<br><code>Integer n = 10;</code><br><b>Unboxing</b>: wrapper → primitivo (automático).<br><code>int x = n;</code>	java::wrapper-classes
Por que não devemos comparar Wrappers com <code>==</code>?	<code>==</code> compara referências, não valores.<br>Funciona de -128 a 127 (cache), mas falha fora disso.<br>Sempre use <code>.equals()</code>.	java::wrapper-classes
Qual a diferença entre <code>ArrayList</code> e <code>LinkedList</code>?	<b>ArrayList</b> (array interno): leitura O(1) — vai direto pelo índice. Inserção/remoção O(n) — precisa empurrar todos os elementos.<br><b>LinkedList</b> (nós encadeados): leitura O(n) — percorre nó a nó. Inserção/remoção O(1) — só reconecta ponteiros.<br><br><b>Dica:</b> Na dúvida, use ArrayList (acesso por índice é mais comum no dia a dia).	java::lists
O que é a Notação Big O?	Forma padrão de descrever a <b>eficiência</b> de uma operação conforme o tamanho dos dados cresce.<br>Responde: "como o tempo dessa operação escala?"	java::big-o
O que significa O(1) (tempo constante)?	A operação é <b>sempre rápida</b>, não importa o tamanho dos dados.<br>10 ou 10 milhões de elementos → mesmo tempo.<br><b>Analogia:</b> abrir um livro na página marcada.	java::big-o
O que significa O(n) (tempo linear)?	A operação fica <b>mais lenta proporcionalmente</b> ao tamanho dos dados.<br>Dobrou os dados → dobrou o tempo.<br><b>Analogia:</b> procurar alguém numa fila — quanto maior, mais demora.	java::big-o
O que significa O(log n) (tempo logarítmico)?	A cada passo, descarta metade dos dados. Muito eficiente mesmo com dados grandes.<br><b>Analogia:</b> busca binária — abrir dicionário no meio e decidir se vai pra esquerda ou direita.	java::big-o
Por que <code>ArrayList.get(i)</code> é O(1)?	Internamente é um array contíguo na memória. O Java calcula o endereço exato do índice <code>i</code> sem precisar percorrer nada.	java::lists java::big-o
Por que <code>ArrayList.add(i, elem)</code> no meio é O(n)?	Ao inserir na posição <code>i</code>, todos os elementos depois de <code>i</code> precisam ser <b>empurrados uma posição</b> para abrir espaço.	java::lists java::big-o
Por que <code>LinkedList.get(i)</code> é O(n)?	Não existe acesso direto. É preciso <b>percorrer nó a nó</b> desde o início (ou fim) até chegar na posição <code>i</code>.	java::lists java::big-o
Por que inserir/remover no início de uma <code>LinkedList</code> é O(1)?	Basta <b>reconectar os ponteiros</b> dos nós vizinhos. Nenhum outro elemento precisa ser movido.	java::lists java::big-o
Qual a forma correta de declarar uma lista em Java?	Declare com a interface, instancie com a implementação:<br><code>List&lt;String&gt; lista = new ArrayList&lt;&gt;();</code><br>Isso permite trocar a implementação sem mudar o resto do código.	java::lists
Como criar uma lista imutável em Java?	<code>List&lt;String&gt; fixa = List.of("A", "B", "C");</code><br>Não é possível adicionar ou remover elementos.	java::lists
Qual a diferença entre <code>HashMap</code>, <code>LinkedHashMap</code> e <code>TreeMap</code>?	<b>HashMap</b>: mais rápido, sem ordem.<br><b>LinkedHashMap</b>: mantém ordem de inserção.<br><b>TreeMap</b>: chaves ordenadas automaticamente.	java::maps
Como iterar um Map em Java?	<pre><code>for (Map.Entry&lt;K,V&gt; entry : mapa.entrySet()) {<br>    entry.getKey();<br>    entry.getValue();<br>}</code></pre>Ou: <code>mapa.forEach((k, v) -&gt; ...)</code>	java::maps
Quais são os 4 pilares da POO?	<b>Abstração</b>: focar no essencial.<br><b>Encapsulamento</b>: esconder detalhes, controlar acesso.<br><b>Herança</b>: reutilizar código entre classes.<br><b>Polimorfismo</b>: tratar objetos diferentes de forma uniforme.	java::poo
Qual a diferença entre interface e classe abstrata em Java?	<b>Interface</b>: múltipla implementação, só constantes, sem construtor, define contrato.<br><b>Classe abstrata</b>: herança única, tem atributos e construtor, compartilha código.	java::poo
O que faz a anotação <code>@Override</code>?	Indica que o método sobrescreve um método da superclasse.<br>O compilador verifica se a assinatura está correta.<br>É boa prática sempre usar.	java::poo
Quais são os modificadores de acesso em Java?	<code>private</code>: só na classe.<br><code>(default)</code>: mesmo pacote.<br><code>protected</code>: pacote + subclasses.<br><code>public</code>: qualquer lugar.	java::poo
Qual a diferença entre <code>extends</code> e <code>implements</code>?	<code>extends</code>: herdar de uma classe (apenas uma).<br><code>implements</code>: implementar interfaces (várias).	java::poo
O que é um Stream em Java?	API para processar coleções de forma <b>declarativa e funcional</b>.<br>Equivalente aos métodos de array do JS/TS (<code>.map()</code>, <code>.filter()</code>, <code>.reduce()</code>).	java::streams
Qual a diferença entre operação intermediária e terminal em Streams?	<b>Intermediária</b>: retorna novo Stream, é lazy (filter, map, sorted).<br><b>Terminal</b>: executa o pipeline e retorna resultado (collect, forEach, reduce, count).	java::streams
Por que Streams são "lazy" (preguiçosos)?	Nada executa até uma <b>operação terminal</b> ser chamada.<br>As operações intermediárias apenas montam o pipeline.	java::streams
Qual o equivalente Java do <code>.find()</code> do TypeScript?	<code>.findFirst()</code> — retorna um <code>Optional</code>.<br><pre><code>Optional&lt;Integer&gt; n = numeros.stream()<br>    .filter(x -&gt; x &gt; 5)<br>    .findFirst();</code></pre>	java::streams
Qual o equivalente Java do <code>.some()</code> do TypeScript?	<code>.anyMatch()</code><br><code>boolean algum = lista.stream().anyMatch(n -&gt; n &gt; 5);</code>	java::streams
Qual o equivalente Java do <code>.every()</code> do TypeScript?	<code>.allMatch()</code><br><code>boolean todos = lista.stream().allMatch(n -&gt; n &gt; 0);</code>	java::streams
Como coletar um Stream de volta para uma List?	<code>.collect(Collectors.toList())</code><br>Ou no Java 16+: <code>.toList()</code>	java::streams
O que são Lambdas em Java?	Funções anônimas. O operador <code>-&gt;</code> é equivalente ao <code>=&gt;</code> (arrow function) do TypeScript.	java::lambdas
Qual operador Java é equivalente ao <code>=></code> do TypeScript?	O operador <code>-&gt;</code> (lambda).<br>TS: <code>(n) =&gt; n * 2</code><br>Java: <code>(n) -&gt; n * 2</code>	java::lambdas
O que é uma interface funcional em Java?	Interface com <b>apenas um método abstrato</b>.<br>Pode receber lambdas como implementação.<br>Ex: <code>Predicate</code>, <code>Function</code>, <code>Consumer</code>.	java::lambdas
Qual a diferença entre <code>Consumer</code>, <code>Function</code> e <code>Predicate</code>?	<code>Consumer&lt;T&gt;</code>: recebe T, retorna void (forEach).<br><code>Function&lt;T,R&gt;</code>: recebe T, retorna R (map).<br><code>Predicate&lt;T&gt;</code>: recebe T, retorna boolean (filter).	java::lambdas
O que é Method Reference (<code>::</code>) em Java?	Atalho para lambda quando ela apenas chama um método.<br><code>s -&gt; s.toUpperCase()</code> → <code>String::toUpperCase</code><br><code>s -&gt; System.out.println(s)</code> → <code>System.out::println</code>	java::lambdas
Quais são os tipos de Method Reference?	<b>Método estático</b>: <code>Math::abs</code><br><b>Método de instância (tipo)</b>: <code>String::toUpperCase</code><br><b>Método de instância (objeto)</b>: <code>System.out::println</code><br><b>Construtor</b>: <code>ArrayList::new</code>	java::lambdas
Qual a diferença entre <code>package</code> e <code>import</code> em Java?	<code>package</code>: declara onde SUA classe está (endereço/namespace).<br><code>import</code>: traz classes de OUTROS pacotes para usar.	java::imports
No Java, por que todo código precisa estar dentro de uma classe?	Java é puramente orientado a objetos. Não existem variáveis ou funções "soltas".<br>Diferente do JS/TS onde <code>export const</code> e funções livres são permitidos.	java::imports
O que é um import estático em Java?	Permite usar métodos/constantes sem qualificar com a classe.<br><code>import static java.lang.Math.PI;</code><br>Uso: <code>double area = PI * r * r;</code> (sem <code>Math.</code>)	java::imports
Qual a forma moderna de fazer requisições HTTP em Java?	<code>HttpClient</code> (Java 11+).<br><pre><code>HttpClient client = HttpClient.newHttpClient();<br>HttpRequest req = HttpRequest.newBuilder()<br>    .uri(URI.create(url)).build();<br>HttpResponse&lt;String&gt; resp = client.send(req,<br>    HttpResponse.BodyHandlers.ofString());</code></pre>	java::http
Java tem suporte nativo a JSON?	<b>Não</b>. Precisa de bibliotecas externas.<br><b>Gson</b> (Google): simples.<br><b>Jackson</b>: mais completo.<br>Diferente do JS que tem <code>JSON.parse()</code> nativo.	java::http
Qual a diferença entre <code>==</code> em Java e TypeScript?	Java: <code>==</code> compara valores primitivos ou referências de objetos.<br>TS: <code>==</code> com coerção de tipo, <code>===</code> comparação estrita.<br>Em Java não existe <code>===</code>.	java::operadores
Java tem operador spread (<code>...</code>), nullish coalescing (<code>??</code>) ou optional chaining (<code>?.</code>)?	<b>Não</b> para nenhum deles.<br>Spread: usar <code>new ArrayList&lt;&gt;(lista)</code>.<br>??: usar ternário ou <code>Optional</code>.<br>?.: usar <code>Optional</code> ou verificações manuais.	java::operadores
O que é o princípio da Responsabilidade Única (SRP)?	Uma classe deve ter <b>apenas uma razão para mudar</b>.<br>Ex: Separar <code>Usuario</code>, <code>UsuarioRepository</code> e <code>EmailService</code> em classes distintas.	java::poo::solid
O que é o princípio Aberto/Fechado (OCP)?	Classes devem estar <b>abertas para extensão</b>, mas <b>fechadas para modificação</b>.<br>Use interfaces e polimorfismo para estender comportamento.	java::poo::solid
O que é Dependency Inversion (DIP)?	Dependa de <b>abstrações</b>, não de implementações concretas.<br>❌ <code>private MySQLDatabase db = new MySQLDatabase();</code><br>✅ <code>private Database db;</code> (interface injetada)	java::poo::solid
